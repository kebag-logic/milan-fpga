## 2. AVTP Audio Format — AAF (Clause 7)

| # | Clause | Required behavior | Module | Verification today / tsn_gen | Why it matters |
|---|--------|-------------------|--------|------------------------------|----------------|
| AAF-1 | 7.2.3 | format field = declared PCM type (INT32 = 0x02 in use) | aaf_talker_i2s (TX), rx_monitor format compare (RX) | ✅ RTL aaf (format golden); avtp_rxmon (UNSUPPORTED_FORMAT counts nothing else) | 7.2.3: interpreting INT32 payload as INT16 is white noise at full scale. |
| AAF-2 | 7.2.4 | sp (sparse timestamp) handling — Milan base formats are non-sparse (tv every PDU) | talker (sp=0), listener | ✅ talker fixed sp=0 (golden byte); RTL avtp_rxmon [29]: a sustained sp=1 stream is DELIBERATELY rejected-and-counted (UNSUPPORTED_FORMAT per PDU, no FRAMES_RX/lock/PCM through parser→monitor→depacketizer) and sp=0 resumes clean | 7.2.4: a sparse talker is legal 1722; our listener's behavior on it is now *deliberate*: count UNSUPPORTED_FORMAT (Milan base formats are non-sparse). |
| AAF-3 | 7.3.2 | nsr field = nominal sample rate (48 kHz = 0x5) | both directions | ✅ RTL aaf (nsr golden); rxmon compare | 7.3.2: nsr is how a listener detects rate mismatch *before* the servo drifts. |
| AAF-4 | 7.3.3 | channels_per_frame declared = channels on the wire | talker (2ch truthful); KL_avtp_rx_monitor exports wire channels; KL_i2s_playback renders per wire truth | ✅ RTL aaf (ch=2); i2spb walker rates for C=1..8 (USER 1-to-1 wire-truth rule); SILICON (declared-8/wire-2 silence root-cause) | 7.3.3: the declared-vs-wire mismatch produced total silence in the field; render now follows the wire, never the AEM store. |
| AAF-5 | 7.3.4 | bit_depth field (32) consistent with payload | both | ✅ RTL aaf (depth 32 golden) | 7.3.4: depth < container width defines MSB-justified padding — mis-read shifts the audio 8 bits (≈ -48 dB or clipping). |
| AAF-6 | 7.3.5 | pcm_data_payload packing: channels interleaved per sample, MSB-justified | aaf_talker_i2s / KL_aaf_rx_depacketizer | ✅ RTL aaf (payload bytes vs SDOUT-tied pattern); i2spb sample-exact ramp decode; SILICON sign-square DOUBLE-Philips fix (loop −4.5 → −69.9 dB) | 7.3.5 + I2S framing is where the sign-bit corruption lived — TB chip models had duplicated the same delay bug, hence the loop-measurement gate. |
| AAF-7 | 7.3.6 | Channel assignment: ch0/ch1 = stereo pair order preserved | depacketizer rotate + playback walker | ✅ RTL i2spb (L/R order, mono {s,0} rule); RTL pcmlpf (L/R independence) | 7.3.6: swapped L/R passes every level test and fails every human. |
| AAF-8 | 7.5 | AAF timestamp = presentation time of the *first* sample in the PDU; listener renders when gPTP reaches it | talker ts capture; i2spb scheduler | ✅ RTL aaf (ts vs sample cadence); SILICON presentation-offset register (AECP-driven transit_ns) | 7.5: off-by-one-sample timestamping accumulates into inter-device echo/flanging in multi-listener rooms. |
| AAF-9 | 7.6 | Packetization cadence: samples_per_frame × rate = class-A 8000 PDU/s (6 samples @ 48 k) | aaf_talker_i2s framer | ✅ RTL aaf (frame cadence w/ media_adv); SILICON wire captures (122,880 ns frame bug found+fixed) | 7.6: the PDU rate is what SRP reserved bandwidth for — over-rate talkers get pruned by the bridge's policer. |
