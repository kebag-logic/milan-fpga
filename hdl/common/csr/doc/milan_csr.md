## Description

Memory-mapped control plane (REQUIREMENTS.md `REQ-CSR-*`) that the Linux driver
binds to. It converts the previously compile-time-only TSN knobs (MAC config,
802.1Q classifier map, 802.1Qav CBS slopes, PTP clock control) into
runtime-writable registers, and exposes MAC statistics, link status and
interrupts back to software. The AXI4-Lite slave uses the combinational-ready,
single-outstanding pattern so a transfer commits on the same rising edge that
`*READY` is asserted (no lost writes when a master drops `*VALID`). Command
strobes (PTP settime/adjtime/snapshot, stats snapshot/reset) are single-cycle
pulses; fields consumed in the `gtx_clk` domain are CDC-synchronised at the
consumer using those strobes (`REQ-CSR-03`).

## Processes

- **axi_resp_fsm**: `always_ff @(posedge aclk)` — AXI response-channel valids
  (BVALID/RVALID); raise on a transfer, clear when accepted.
- **register_write**: `always_ff @(posedge aclk)` — register file write path:
  synchronous reset defaults, hardware event latching (before W1C), AXI-Lite
  register writes, W1C on IRQ_STATUS, single-cycle command strobes, and the
  per-queue CBS window.
- **read_mux**: `always_comb` — combinational read address decode.
- **read_data_reg**: `always_ff @(posedge aclk)` — latch read data on the AR
  handshake so RDATA is stable while RVALID is held.
- **gen_cbs_out**: `generate for` — flatten the per-queue CBS registers onto the
  packed output buses.
